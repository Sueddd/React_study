1) ref는 값을 유지하고 값을 변경해도 랜더링을 해주지 않는다.
state를 통해 랜더링을 해준다해도 값을 유지하기 때문에 초기화가 되지 않는다. 

2) state는 값을 변경할 때마다 랜더링이 진행된다. 
값을 유지하면서, 랜더링을 해주는 것이기 때문에 
앱 컴포넌트 안의 함수들이 다 다시 실행되도 값이 초기화 되지 않고
이전의 값을 그대로 유지 

3) var, const, let 일반 변수는 값을 유지하지도, 랜더링을 하지도 않는다. 
변수 값을 변경하고 랜더링을 해주면 다시 초기화가 되어 
초기값으로 돌아가기 때문에 일반 변수는 자료를 잠깐 저장할 때 
사용하는 것이 유용하다.
